========================================================================
ASSETSYNC - FRONTEND PROJECT CONTEXT
========================================================================

Welcome to the AssetSync frontend project! This document provides an overview
of the project's purpose, technology stack, current structure, implemented
features, and development guidelines for the team.

------------------------------------------------------------------------
1. PROJECT OVERVIEW & TECH STACK
------------------------------------------------------------------------
AssetSync is a high-fidelity, responsive frontend web application designed
to manage university resources (physical equipment and locations/rooms).

Core Technologies:
- Framework: React 19 (JavaScript ES module structure)
- Build Tool: Vite
- UI Library: Material UI (MUI v9)
- Styling: Custom MUI CSS-in-JS + Theme Provider (Premium Dark Mode)
- HTTP Client: Axios for API integrations
- Development Tools: ESLint, Prettier

------------------------------------------------------------------------
2. DIRECTORY STRUCTURE
------------------------------------------------------------------------
The codebase is structured logically under the `src/` directory:

src/
├── assets/          - Static assets (images, logos, etc.)
├── components/      - Reusable UI elements
│   └── Navbar.jsx   - Sticky glassmorphic navigation bar with tab switching and role indicator
├── services/        - API helper services
│   └── api.js       - Service layer for backend REST API calls
├── views/           - Full page/tab views
│   ├── Login.jsx    - Interactive role selection landing page
│   └── ResourceCatalog.jsx - Fully featured asset catalog display
├── App.css          - Global application CSS overrides
├── App.jsx          - Root component housing theme config, main layout, and navigation
├── index.css        - Global baseline CSS
└── main.jsx         - React application entrypoint

------------------------------------------------------------------------
3. CURRENTLY IMPLEMENTED FEATURES
------------------------------------------------------------------------

A. Custom Dark Theme & Premium Aesthetics (App.jsx)
- Custom Material-UI theme featuring a rich, deep blue-slate dark mode.
- Subtle background gradients, glassmorphism panel styles, and responsive layout.
- Modern custom typography leveraging 'Inter' and 'Outfit' Google Fonts.

B. Asset Catalog (views/ResourceCatalog.jsx)
- Integrated Data Fetching: Calls `fetchResources()` API helper to load assets
  from the backend, including loading indicators and robust error-handling.
- Dynamic Tab Filtering: Toggle bar allowing users to filter by "All Assets",
  "Locations", or "Equipment".
- Client-Side Live Search: Real-time filtering by asset name or category.
- Highly Interactive Grid:
  - Locations display capacity information (e.g., student capacity).
  - Equipment displays quantities and condition/quality indicators.
  - Quality conditions (Good, New, Maintenance, Needs Inspection) are
    dynamically color-coded with subtle glowing status dots.
  - Premium micro-interactions: cards scale upwards slightly and glow purple
    on hover.

C. Equipment Booking (views/EquipmentBooking.jsx) (Active)
- Real-time Equipment Booking: Form interface accessible on Tab 1 ("Book Equipment").
- Form Fields: `resourceId` (selected via Autocomplete search suggestions showing name, category, and live available quantity), `requesterName` (text input), `requestedQuantity` (number input), `startTime` (datetime-local), and `endTime` (datetime-local). User role selection dropdown has been removed.
- Role Integration: Automatically injects the active user role (Lecturer/Student) from the login session into the booking payload.
- Role Badge: Displays a subtle colored gradient badge (e.g. violet for Lecturer, indigo for Student) highlighting the active booking profile at the top of the form.
- Feedback Handling:
  - Submit Button: Prominent button with a gradient of purple-indigo and a loading spinner while API booking is in flight.
  - Successful booking (200 OK) displays a green Snackbar Alert: "Success: Equipment Allocated." and clears the form.
  - Insufficient inventory or resource not found (409 Conflict / 404 Not Found) displays a red Alert with the exact message from the backend.

D. Location Batch Allocator (views/LocationBooking.jsx, views/BatchProcessor.jsx) (Active)
- Venue Advance Request Form: Form allowing users to submit location requests. Fields include category ("Lecture Hal", "Computer Lab", "Auditorium"), requester name, user role (automatically mapped to priority: 1 for Lecturer, 2 for Student), estimated student capacity, and start/end times. Displays feedback Snackbar.
- System Admin Batch Allocator Panel: Prominent, centralized control card to manually run the T-24 allocation algorithm for demo purposes. Includes disabled button states, spinner indicators during execution, and success Alert messages.

E. Master Calendar View (views/MasterCalendar.jsx) (Active)
- The shell implements a 4-tab dashboard structure. All tabs are now active:
  - Tab 0: Catalog (Active)
  - Tab 1: Book Equipment (Active)
  - Tab 2: Location Batch Allocator (Active - houses sub-navigation toggle to switch between Request Form and Admin Panel)
  - Tab 3: Master Calendar (Active - aggregates location allocations and equipment bookings into a unified schedule view)

F. Role-Selection Landing Page (views/Login.jsx) (Active)
- High-fidelity simulated login interface presented to unauthenticated users.
- Prompts users to select their role: "Lecturer" or "Student" using premium hover-animated card buttons featuring styled SVG icons.
- Selected role persists in `localStorage` to retain the session across reloads.
- Includes a prominent "Switch Role" button in the navigation bar to sign out/change roles instantly.

4. BACKEND INTEGRATION
------------------------------------------------------------------------
The frontend connects to a backend API running on `http://localhost:8080/api`.
The current integration focuses on:
- Endpoint: GET `/api/resources`
  - Query Params: `type` (optional, can be 'LOCATION' or 'EQUIPMENT')
- Endpoint: POST `/api/equipment/book`
  - Payload: booking details for hardware checkout.
- Endpoint: POST `/api/locations/request`
  - Payload: location booking request details (mapped categories, priorities).
- Endpoint: POST `/api/locations/process-batch`
  - Triggers the 24-hour batch scheduling allocation algorithm.
- Endpoint: GET `/api/calendar/events`
  - Returns: Sorted combined array of location allocations and equipment bookings.


------------------------------------------------------------------------
5. DEVELOPMENT GUIDELINES
------------------------------------------------------------------------
- Maintaining Aesthetics: Ensure new components follow the existing premium,
  glassmorphic theme. Avoid using default, unstyled elements or introducing
  discordant color schemes.
- Light Dependencies: Use self-contained visual elements (such as the SVG paths
  defined in `ResourceCatalog.jsx` for search/users/box icons) to keep bundle size low
  and reduce third-party package reliance.
